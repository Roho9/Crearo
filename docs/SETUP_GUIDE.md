# SETUP_GUIDE.md — From zero to a running Crearo build

A practical, ordered checklist to go from this repo to a signed app on a device, with the backend wired. Assumes macOS + an iPhone for device testing.

---

## 0. Prerequisites (install these first)

```bash
# Xcode (from the App Store) + command line tools
xcode-select --install

# Homebrew (https://brew.sh) then the tooling we use
brew install xcodegen swiftlint git supabase/tap/supabase
# (Node is bundled with the Supabase CLI workflow; install if you want it standalone)
brew install node
```

- **Xcode 15.4+** (iOS 17 SDK: needs `RealityView`, SwiftData, the Observation framework).
- **XcodeGen** — generates `Crearo.xcodeproj` from `project.yml` so the project file isn't a merge-conflict nightmare in Git.
- **SwiftLint** — style/consistency.
- **Supabase CLI** — local backend + deploying Edge Functions.

---

## 1. Apple Developer account & signing

1. **Enroll** in the [Apple Developer Program](https://developer.apple.com/programs/) ($99/yr). Personal builds work with a free Apple ID, but you need the paid program for TestFlight, push, CloudKit production, and the App Store.
2. In **App Store Connect**, create an app record (bundle id e.g. `com.yourstudio.crearo`).
3. **Capabilities** to enable on the App ID (Certificates, Identifiers & Profiles, and in Xcode → Signing & Capabilities):
   - **Sign in with Apple** (auth, GDD §61).
   - **iCloud → CloudKit** (optional private media, §TECH 5) — creates a CloudKit container.
   - **Push Notifications** (later: daily-quest/decay nudges — used gently, GDD §31).
4. **Signing:** easiest is Xcode **Automatically manage signing** with your Team selected. For CI, create an **App Store Connect API key** (used by Fastlane/match or xcodebuild) and store it as a CI secret.
5. Add your test device's UDID to the provisioning profile (automatic signing handles this when the device is connected).

## 2. Git

```bash
cd Crearo
git init
git add .
git commit -m "Crearo: design docs + iOS scaffolding + backend"
git branch -M main
# create an empty repo on GitHub/GitLab, then:
git remote add origin git@github.com:yourstudio/crearo.git
git push -u origin main
```

- `.gitignore` (in the repo root) already excludes Xcode user data, build artifacts, `*.xcodeproj` (regenerated by XcodeGen), `.env`, and secrets.
- **Never commit secrets.** API keys live in `.env` (gitignored) and in CI secrets / Edge Function env (see §5–6).
- Suggested branch flow: `main` (protected) ← PRs from `feature/*`; CI must pass (`swift test` + `xcodebuild test`).

## 3. Generate & open the Xcode project

```bash
cd Crearo
xcodegen generate          # reads project.yml → produces Crearo.xcodeproj
open Crearo.xcodeproj
```

- The project has two targets: **CrearoApp** (the iOS app) and **CrearoCore** (the Swift package dependency, also independently testable).
- Set your **Team** and a unique **bundle identifier** in Signing & Capabilities.
- Build & run on a simulator first, then a device.

> If you prefer not to use XcodeGen, you can create a new iOS App project in Xcode manually and drag the `CrearoApp/` sources in, then add `CrearoCore` as a local Swift Package (File → Add Package Dependencies → Add Local…). XcodeGen just automates this reproducibly.

---

## 4. Run CrearoCore tests without a device (fast loop)

`CrearoCore` is pure Swift (Foundation-only), so it builds and tests on the command line / Linux CI:

```bash
cd CrearoCore
swift build
swift test          # runs CreativityScoringEngineTests, BalanceEngineTests, EconomyTests
```

This is your tight TDD loop for game logic — no simulator needed.

## 5. Backend — Supabase (recommended)

1. Create a project at [supabase.com](https://supabase.com) (or run locally: `supabase init && supabase start`).
2. **Enable pgvector & apply schema:**
   ```bash
   cd supabase
   supabase db push           # or paste schema.sql into the SQL editor
   ```
   `schema.sql` creates the `profiles`, `creations`, `resource_ledger`, `prompts`, and `prompt_responses` tables, enables `vector`, builds the **HNSW** index, and sets **Row-Level Security**.
3. **Auth:** in Supabase Auth settings, enable **Apple** provider; paste your Apple Service ID + key (from the Apple Developer portal, Sign in with Apple config).
4. **Edge Functions** (AI brokering, scoring §6):
   ```bash
   supabase functions deploy interpret-idea
   supabase functions deploy score-originality
   # set model keys as function secrets (NEVER in the app):
   supabase secrets set OPENAI_API_KEY=sk-...        # or your LLM/embeddings provider
   ```
5. **App config:** put your Supabase URL + anon key in `CrearoApp/Resources/Config.xcconfig` (gitignored) — *only* the anon key ships in the app; it's protected by RLS. Service-role keys live **only** in Edge Functions.

### Alternative: Firebase
If you choose Firebase instead, implement `FirebaseAuthService` / `FirebasePersistence` (conform to the same protocols in `CrearoCore/Services`), add `GoogleService-Info.plist` (gitignored), and move the AI brokering to Cloud Functions. Note: you'll need a separate vector solution for the rarity engine (Vertex AI Vector Search or a pgvector instance) — this is why Supabase is the default.

### Optional: CloudKit for private media
Enable the iCloud/CloudKit capability (§1), create a container, and implement `CloudKitMediaStore` for drawings/photos/voice so intimate media stays in the user's private iCloud DB.

## 6. Secrets & configuration summary

| Secret | Lives in | Never in |
|--------|----------|----------|
| LLM / embedding API keys | Supabase Edge Function secrets (server) | the app binary, Git |
| Supabase **service-role** key | Edge Functions / server env | the app, Git |
| Supabase URL + **anon** key | `Config.xcconfig` (gitignored), shipped in app (RLS-protected) | committed files |
| Apple Sign in keys | Apple portal + Supabase Auth config | the app, Git |
| App Store Connect API key | CI secret store | the repo |

## 7. Continuous Integration (GitHub Actions sketch)

`.github/workflows/ci.yml` (create when you push to GitHub):
```yaml
name: ci
on: [push, pull_request]
jobs:
  core-linux:                 # fast: pure logic, no simulator
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: swift-actions/setup-swift@v2
      - run: cd CrearoCore && swift test
  app-macos:
    runs-on: macos-14
    steps:
      - uses: actions/checkout@v4
      - run: brew install xcodegen
      - run: xcodegen generate
      - run: xcodebuild test -scheme CrearoApp -destination 'platform=iOS Simulator,name=iPhone 15'
```

## 8. First-run smoke test (what "working" looks like for the MVP)

1. App launches → **Sign in with Apple** → a profile row is created in Supabase.
2. **Opening sequence** (GDD §10): wake → light fire → make a mark → name the companion.
3. Open the **Creation Forge** → type *"a sword that shoots honey to slow enemies"* → `interpret-idea` returns **Honeyfang** with clamped stats → it appears, named, in your inventory and on a home display.
4. The **resource wallet** decremented; a **prophecy line** appears.
5. Force-quit, relaunch → your creation, companion name, and home state **persist** (local + cloud).
6. `swift test` in `CrearoCore` is green.

If those six work, the core loop and the engine boundary are proven — that's the MVP skeleton this repo targets.
